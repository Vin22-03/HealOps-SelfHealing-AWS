About HealOps

HealOps is a hands-on SRE & DevOps project built with a failure-first mindset. Instead of assuming systems never fail, it demonstrates how real cloud systems detect failures, trigger recovery, and measure MTTR.

What this project proves

HealOps shows the complete incident lifecycle using real AWS behavior. For example, when CPU usage spikes or a container becomes unhealthy, CloudWatch alarms and ECS task state change events are emitted. These signals are captured via EventBridge and recorded as incidents.

When an ECS task stops or becomes unhealthy, the ECS scheduler automatically replaces the task to maintain the desired count. HealOps observes this recovery and calculates MTTR (Mean Time To Recovery) based on actual timestamps â€” not assumptions.

Nothing here is simulated: failures are intentional, detection is event-driven, healing is handled by AWS, and recovery time is measured from real signals.
