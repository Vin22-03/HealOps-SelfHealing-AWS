About HealOps

A recruiter-facing SRE/DevOps showcase built with a failure-first mindset. The goal is not to pretend “magic self-healing” — the goal is to demonstrate how real systems detect failure and recover, and how we quantify MTTR.

What this project proves

HealOps demonstrates incident lifecycle thinking: you can see what broke, when it broke, how it was detected, what healed it, and how long recovery took. The dashboard is intentionally designed so the data source can evolve: today it reads demo incidents; later it can ingest real AWS signals without rewriting the UI.

This is audit-friendly and honest by design: no unsafe chaos buttons, no fake “auto-heal” claims — only observed reality.

Architecture

  • AWS ECS (Fargate) — service scheduler for task replacement
  • Application Load Balancer — health checks + traffic routing
  • Terraform (IaC) — repeatable infra
  • Jenkins CI/CD — automation-first provisioning & deployment
  • FastAPI UI — dashboard + APIs (demo now, live later)
